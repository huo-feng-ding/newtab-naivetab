# ☁️ 同步与备份

> 多设备同步配置，支持导出文件自行备份

## 🔄 跨设备云同步

NaiveTab 的配置数据利用浏览器内置的 `chrome.storage.sync` 进行同步。

### 同步流程

```
┌─────────────────┐       ┌──────────────┐       ┌─────────────────┐
│  设备 A (PC)     │  ──→  │  浏览器云同步  │  ──→  │  设备 B (笔记本)  │
│                  │       │              │       │                  │
│  修改配置 ───→   │       │  自动同步     │       │  ←── 自动拉取    │
└─────────────────┘       └──────────────┘       └─────────────────┘
```

### 前置条件

- 需要在浏览器中**登录个人账号**（Chrome/Edge/Firefox 账号）
- 开启了浏览器账号同步功能
- 同步由浏览器自动完成，无需额外操作

### 同步的内容

| 同步项 | 说明 |
|--------|------|
| 组件布局 | 各组件在页面上的位置 |
| 组件设置 | 各组件的自定义配置（颜色、字体、大小等） |
| 全局设置 | 背景图、外观模式等全局配置 |
| 备忘录内容 | 备忘录中编辑的文本 |
| 倒计时状态 | 正在运行的倒计时进度 |

### ⚠️ 同步限制

- 浏览器同步存储有容量上限（通常约 100KB）
- 当配置数据（如键盘书签）接近上限时，同步可能失败
- 建议减少书签数量或使用本地备份替代

## 💾 本地备份

### 导出设置

1. 在空白处右键 → **通用设置**
2. 找到 **同步设置** 区域
3. 点击 **「导出」** 按钮
4. 选择保存位置，保存为 `.json` 文件

### 导入设置

1. 在空白处右键 → **通用设置**
2. 找到 **同步设置** 区域
3. 点击 **「导入」** 按钮
4. 选择之前导出的 `.json` 文件

### 适用场景

| 场景 | 推荐方式 |
|------|----------|
| 更换电脑 | 导出 → 新电脑导入 |
| 浏览器重装 | 导出 → 重装后导入 |
| 分享给他人 | 导出 → 发送文件 → 对方导入 |
| 定期备份 | 定期导出保存，防止数据丢失 |

## 🔧 重置设置

| 重置类型 | 说明 | 保留数据 |
|----------|------|----------|
| **快速重置** | 重置设置项到默认值 | ✅ 保留备忘录、倒计时、本地图片等用户数据 |
| **完全重置** | 回到出厂默认状态 | ❌ 清空所有数据，包括用户内容 |

### 操作步骤

1. 在空白处右键 → **通用设置**
2. 找到 **重置设置** 区域
3. 选择 **快速重置** 或 **完全重置**
4. 确认后执行

> ⚠️ 重置前建议先导出备份，以防误操作丢失配置。
